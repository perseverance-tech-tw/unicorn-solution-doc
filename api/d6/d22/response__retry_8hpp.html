Detailed Description

Declares a helper function to display a modal "Retry/Close" dialog.

This header provides the declaration for the showRetryDialog function, a utility for creating a simple, blocking error dialog with options for the user to retry an action or close the dialog.

◆ GTK_RESPONSE_RETRY

#define GTK_RESPONSE_RETRY   8

Defines a custom GTK response ID for a "Retry" button.

Since GTK does not have a predefined GTK_RESPONSE_RETRY, this macro defines a unique integer value (8) to represent the user clicking the "Retry" button in a dialog, allowing it to be easily identified in signal handlers or after gtk_dialog_run.

Displays a modal error dialog with "Retry" and "Close" options.

This function simplifies the process of presenting a blocking error message to the user. It creates a GtkMessageDialog that is modal to the parent window, forcing the user to choose to either retry the failed operation or close the notification.

Parameters
parentA pointer to the parent GtkWindow. The dialog will be modal to this window.
messageThe error message to display inside the dialog.
Returns
bool Returns true if the user clicks the "Retry" button, and false if they click the "Close" button or otherwise close the dialog.

This function creates a GtkMessageDialog, adds custom "Retry" and "Close" buttons with their respective response IDs, and runs the dialog modally. It blocks execution until the user makes a selection. The function returns a boolean indicating whether the "Retry" option was chosen.
