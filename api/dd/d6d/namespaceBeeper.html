Detailed Description

Manages the playback of audible alert sounds (beeps) in a thread-safe manner.

This utility provides a simple API to trigger different beep patterns for various alert zones (e.g., "REGION_FRONT"). It handles the spawning of threads for non-blocking audio playback, ensures thread safety with a mutex, and automatically selects an available system audio player.

The main API function to trigger or update an audible alert for a specific zone.

This function is the primary entry point for using the Beeper. Based on the pat (pattern) provided:

  • NONE: Ensures any existing loop for the key is stopped.
  • BEEP_2_ONCE: Spawns a new, detached thread to play a non-blocking two-beep burst.
  • BEEP_3_LOOP: If not already running, spawns a persistent thread for the key that plays a three-beep burst repeatedly until stop(key) is called.
    Parameters
    keyA unique string identifier for the alert source (e.g., "REGION_FRONT", "REGION_RIGHT").
    patThe desired VRUDetector::BeepPattern to play.
    cfgAn optional Config struct to customize timing for this specific call.
