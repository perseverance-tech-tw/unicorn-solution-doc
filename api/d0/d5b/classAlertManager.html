◆ settleAndRender()

void AlertManager::settleAndRender ( bool  allowLowFrontBorder,
bool  allowLowRightBorder 
)

Evaluates the current risk states and dispatches the appropriate alerts.

Evaluates the current risk states and dispatches the appropriate alerts based on priority.

This is the core logic function, intended to be called once per frame after all zones have been updated via updateZone. It applies a priority system (HIGH > MED > LOW) to decide which popup, beep, or border alert to render, ensuring only the most critical notification is presented.

Parameters
allowLowFrontBorderA boolean indicating if a low-risk condition exists for the front border.
allowLowRightBorderA boolean indicating if a low-risk condition exists for the right border.

This core logic function applies the following rules:

  1. HIGH Risk: If any zone is HIGH, a modal popup is shown for that zone. All beeps are silenced, and no LOW/MED risk borders are displayed.
  2. MED Risk: If no zone is HIGH but at least one is MED, a one-time beep is triggered for the MED zone(s). LOW risk borders are only allowed to be shown in zones that also have a MED risk.
  3. LOW Risk: If no zone is HIGH or MED, all beeps are stopped. LOW risk borders are permitted to be shown based on the input parameters.
    Parameters
    allowLowFrontBorderA boolean indicating if a low-risk condition exists for the front border.
    allowLowRightBorderA boolean indicating if a low-risk condition exists for the right border.
