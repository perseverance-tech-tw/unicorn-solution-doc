Detailed Description

Defines a policy system for mapping frame names to alert zones.

This header-only file provides a simple, rule-based system (resolvePolicyFor) to determine which alert zone (e.g., FRONT, RIGHT) a specific display frame (like "Anypoint 1") corresponds to. This is used by the AlertManager to decide where to apply risk alerts.

◆ resolvePolicyFor()

ZonePolicy resolvePolicyFor ( const std::string &  frameName)
inline

Resolves a ZonePolicy for a given frame name based on predefined rules.

This function acts as a mapping utility. It takes the name of a display frame (e.g., "Mois Anypoint 2") and determines which alert zone it belongs to and whether it is allowed to show low-risk alert borders.

The rules are as follows:

  • Any frame name containing "panorama" is assigned to Zone::NONE and disallows borders.
  • Any frame name containing "anypoint 2" is assigned to Zone::FRONT.
  • Any frame name containing "anypoint 1", "anypoint 3", or "anypoint 4" is assigned to Zone::RIGHT.
  • All others fall back to a default policy (currently Zone::RIGHT).
Parameters
frameNameThe name of the frame record to resolve.
Returns
