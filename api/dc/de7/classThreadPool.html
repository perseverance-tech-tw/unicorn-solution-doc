Detailed Description

Manages a collection of worker threads to execute tasks concurrently.

This class creates a specified number of threads upon construction and provides an enqueue method to submit tasks. Tasks are placed in a queue and are executed by the next available worker thread. The pool ensures a graceful shutdown in its destructor by joining all threads.

ThreadPool::~ThreadPool ( )

Destroys the ThreadPool.

Signals all worker threads to stop, wakes them up, and waits for them to finish their current tasks and terminate by joining them.

This destructor ensures a graceful shutdown of the thread pool. It sets a stop flag, notifies all waiting worker threads to wake up, and then joins each thread to ensure it has completed its execution before the object is destroyed.

Submits a new task to be executed by a worker thread.

This method wraps the provided function into a packaged task, adds it to the task queue, and notifies one of the waiting worker threads. It returns a std::future that can be used to retrieve the function's return value when it completes.
